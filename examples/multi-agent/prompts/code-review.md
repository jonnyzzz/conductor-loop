# Task: Code Review

Please perform a thorough code review of the Python script in `sample-code.py`.

## Review Criteria

Analyze the code for:

1. **Code Quality**
   - Readability and maintainability
   - Naming conventions
   - Code organization
   - Documentation quality

2. **Potential Bugs**
   - Logic errors
   - Edge cases not handled
   - Type errors or type safety issues
   - Off-by-one errors

3. **Performance Issues**
   - Algorithmic inefficiency
   - Unnecessary operations
   - Resource usage problems
   - Scalability concerns

4. **Security Vulnerabilities**
   - Input validation issues
   - SQL injection risks
   - Command injection risks
   - Authentication/authorization problems

5. **Best Practices**
   - Language-specific conventions (PEP 8 for Python)
   - Error handling
   - Testing considerations
   - Dependency management

## Output Format

Write your review to `output.md` with this structure:

```markdown
# Code Review: sample-code.py

## Executive Summary
[Brief overview of code quality and main issues]

## Critical Issues
[Issues that must be fixed]

## Warnings
[Issues that should be fixed]

## Suggestions
[Nice-to-have improvements]

## Positive Aspects
[Things the code does well]

## Recommendations
[Prioritized action items]
```

## Sample Code Location

The file to review is located at: `sample-code.py` in the current directory.

## Instructions

1. Read and analyze the sample code thoroughly
2. Identify issues in all categories listed above
3. Provide specific line numbers and code snippets
4. Suggest concrete improvements with examples where applicable
5. Write your complete review to `output.md`
